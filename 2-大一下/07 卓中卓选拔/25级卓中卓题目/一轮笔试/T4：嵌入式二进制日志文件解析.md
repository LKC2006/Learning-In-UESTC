# T4：嵌入式二进制日志文件解析

某嵌入式设备在工作过程中，将传感器采样数据持续写入一个自定义格式的二进制日志文件，文件扩展名为 `.bvt`。你需要使用 **C 语言** 开发一个工具，该工具首先将整个 `.bvt` 文件 **加载到内存**，然后在内存中对二进制数据进行 **解析、校验、查询、修改**，所有核心操作（校验、查询、修改）均基于内存缓冲区，不得在操作过程中频繁读写文件。

### 文件格式

所有多字节整数采用 **小端** 字节序（Little‑Endian）。结构体在内存中可能因对齐而产生空隙，读写时必须显式按字节处理，严禁依赖编译器填充，即结构体紧凑对齐（使用 `#pragma pack(1)`）。

### 文件头（固定 32 字节）

| 偏移 (byte) | 长度 (byte) | 字段名          | 类型        | 描述                                                      |
| ----------- | ----------- | --------------- | ----------- | --------------------------------------------------------- |
| 0           | 4           | magic           | char[4]     | 魔数，固定为 `{'B','V','T','1'}`                          |
| 4           | 4           | version         | uint32_t    | 文件格式版本号，当前为 1                                  |
| 8           | 4           | record_count    | uint32_t    | 文件中包含的记录总数                                      |
| 12          | 16          | reserved        | uint8_t[16] | 保留，全零                                                |
| 28          | 4           | header_checksum | uint32_t    | 文件头校验和：对前 28 字节（0~27）按字节累加和（模 2^32） |
|             |             |                 |             |                                                           |

### 数据记录（每条固定 32 字节）

| 偏移 (byte) | 长度 (byte) | 字段名          | 类型        | 描述                                                         |
| ----------- | ----------- | --------------- | ----------- | ------------------------------------------------------------ |
| 0           | 4           | timestamp       | uint32_t    | Unix 时间戳（秒）                                            |
| 4           | 2           | sensor_id       | uint16_t    | 传感器 ID（1~65535）                                         |
| 6           | 1           | sensor_type     | uint8_t     | 传感器类型：0 = 温度，1 = 湿度，2 = 气压                     |
| 7           | 4           | value           | float       | 采样数值（IEEE 754 单精度）                                  |
| 11          | 1           | flags           | uint8_t     | 标志位（bit0：有效标志，1=有效 0=无效；bit1：校正标志，1=已校正） |
| 12          | 19          | reserved        | uint8_t[19] | 保留，全零                                                   |
| 31          | 1           | record_checksum | uint8_t     | 记录校验和：对前 31 字节（0~30）的每个字节的累加和，取低 8 位（模256） |

### 请实现以下函数

1. 解析与校验

   ```c
   int parse_and_validate(const uint8_t *data, size_t size, int *record_count_out);
   ```

   校验魔数、版本号、文件头校验和。

   校验每条记录的校验和。

   返回值：0 表示全部正确；1 表示文件头错误（魔数/版本/头校验和错）；2 表示至少有一条记录校验和错误；-1 表示数据长度不足或其他错误。

2. 查询记录

   ```c
   int query_records(const uint8_t *data, size_t size,
                  int sensor_type, uint32_t start_ts, uint32_t end_ts,
                  uint8_t *out_records, int max_records);
   ```

   筛选出满足条件的记录：

   `sensor_type`：传感器类型，`-1` 表示不过滤，否则匹配 `sensor_type` 字段。

   `start_ts`、`end_ts`：时间戳范围（闭区间），只保留 `timestamp` 在 `[start_ts, end_ts]` 内的记录。

   将符合条件的记录**按原顺序**复制到 `out_records` 指向的数组中，数组最多能容纳 `max_records` 条记录。

   函数返回实际查询到的记录数量（可能小于等于 `max_records`）。

   若发生错误（如 `data` 无效、文件头损坏等），返回 `-1`。

3. 修改记录修改指定索引记录的 `value` 字段，并重新计算该记录的校验和。返回值同上。

   ```c
   int modify_record_flags(uint8_t *data, size_t size, uint32_t index, uint8_t new_flags);
   ```

   修改指定索引记录的 `flags` 字段，并重新计算该记录的校验和（更新内存中的 `record_checksum`）。

   成功返回 0；索引越界返回 -2；文件头格式错误或内存不足返回 -1。

   ```c
   int modify_record_value(uint8_t *data, size_t size, uint32_t index, float new_value);
   ```
