# T3：三地址代码基本块优化器

实现一个C语言程序，对给定的三地址代码（仅包含一个基本块）进行经典的基本块内优化。输入保证不包含任何跳转指令和标号，因此无需用户手动划分基本块。要求实现以下优化：

常量传播

拷贝传播

常量折叠

代数化简

死代码删除

优化应反复进行，直到所得代码不再发生变化为止。最终输出优化后的三地址代码序列。

### 三地址码说明

1. 三地址码的基本形式 x = y op z，其中：x 是目标变量；y 和 z 是操作数；op 是二元运算符，如：+ - * /。例如：

   ```
   t1 = a + b
   t2 = t1 * 3
   x = t2
   ```

2. 赋值指令： x = y

3. 输出指令：print x，表示输出变量或常量 x 的值。被 print 使用的变量不能被删除。

本题中的常量均为十进制整数，变量名由字母、数字和下划线组成，并且必须以字母或下划线开头；所有运算均为整数运算。为了简化问题，测试数据保证：不会出现除数为 0 的情况；所有中间结果都在 32 位有符号整数范围内；输入格式合法，不需要处理语法错误。

### 优化方法说明

本题要求实现以下五种优化。

------

1. 常量传播: 如果某个变量已经被赋值为一个确定的常量，那么在它被重新赋值之前，后续使用该变量的地方可以直接替换为该常量。

### 示例

优化前：

```
a = 10
b = a + 5
print b
```

因为 `a` 的值是常量 `10`，所以可以将 `b = a + 5` 优化为：

```
a = 10
b = 10 + 5
print b
```

进一步进行常量折叠后：

```
a = 10
b = 15
print b
```

------

重新赋值会使原常量失效

优化前：

```
a = 10
b = a + 1
a = x
c = a + 1
print c
```

其中：b = a + 1 中的 `a` 可以替换为 `10`。但是：a = x 之后，`a` 不再是确定常量，因此：c = a + 1 中的 `a` 不能继续替换为 `10`。

------

1. 拷贝传播：如果有赋值语句 a = b， 并且在后续代码中，`a` 和 `b` 的值都没有发生改变，那么可以把后续使用 `a` 的地方替换为 `b`。

### 示例

优化前：

```
a = b
c = a + 1
print c
```

可以优化为：

```
a = b
c = b + 1
print c
```

如果后续 `a = b` 这条赋值语句不再被使用，则可以进一步通过死代码删除去掉：

```
c = b + 1
print c
```

------

### 被重新赋值后拷贝关系失效

优化前：

```
a = b
c = a + 1
b = 5
d = a + 1
print d
```

其中：c = a + 1，可以将 `a` 替换为 `b`。但是：b = 5 之后，原来的 `a = b` 拷贝关系失效，因为 `b` 的值已经改变。因此：d = a + 1 中的 `a` 不能再替换为 `b`。

------

1. 常量折叠: 如果某条二元运算指令的两个操作数都是常量，则可以在编译阶段直接计算结果。

### 示例

优化前：

```
t1 = 2 + 3
t2 = 10 * 4
t3 = 8 - 5
t4 = 20 / 4
```

优化后：

```
t1 = 5
t2 = 40
t3 = 3
t4 = 5
```

------

### 与常量传播结合

优化前：

```
a = 6
b = a * 7
print b
```

先进行常量传播：

```
a = 6
b = 6 * 7
print b
```

再进行常量折叠：

```
a = 6
b = 42
print b
```

------

1. 代数化简：根据代数恒等式，对表达式进行简化。

------

### 加法规则

```
x = y + 0  =>  x = y
x = 0 + y  =>  x = y
```

示例：

```
t1 = a + 0
t2 = 0 + b
```

优化后：

```
t1 = a
t2 = b
```

------

### 减法规则

```
x = y - 0  =>  x = y
```

示例：

```
t1 = a - 0
```

优化后：

```
t1 = a
```

注意：

```
x = 0 - y
```

不能化简为：

```
x = y
```

因为 `0 - y` 与 `y` 语义不同。

------

### 乘法规则

```
x = y * 1  =>  x = y
x = 1 * y  =>  x = y
x = y * 0  =>  x = 0
x = 0 * y  =>  x = 0
```

示例：

```
t1 = a * 1
t2 = 1 * b
t3 = c * 0
t4 = 0 * d
```

优化后：

```
t1 = a
t2 = b
t3 = 0
t4 = 0
```

------

### 除法规则

```
x = y / 1  =>  x = y
```

示例：

```
t1 = a / 1
```

优化后：

```
t1 = a
```

注意：

```
x = 0 / y
```

虽然在数学上当 `y != 0` 时结果为 `0`，但为了避免涉及除零语义和运行时行为，本题不要求对 `0 / y` 做代数化简。

------

1. 死代码删除：如果一条赋值或运算指令定义的变量在后续代码中从未被使用，那么这条指令就是死代码，可以删除。

### 示例

优化前：

```
t1 = a + b
t2 = c + d
print t1
```

`t1` 被 `print t1` 使用，因此不能删除。

`t2` 后续从未被使用，因此可以删除。

优化后：

```
t1 = a + b
print t1
```

------

### 赋值覆盖导致死代码

优化前：

```
a = 1
a = 2
print a
```

第一条：

```
a = 1
```

的结果还没有被使用，就被：

```
a = 2
```

覆盖了，因此第一条赋值是死代码。

优化后：

```
a = 2
print a
```

------

### print 指令不能删除

优化前：

```
a = 10
print a
```

即使 `print a` 后没有其他代码，`print` 仍然具有输出作用，不能删除。

### 优化顺序要求

为了保证输出结果唯一，便于自动评测，本题规定必须按照以下方式进行优化。对整个基本块重复执行以下步骤，直到代码不再发生变化：

```
1. 常量传播
2. 拷贝传播
3. 常量折叠
4. 代数化简
5. 死代码删除
```

也就是说，一轮优化结束后，如果代码发生变化，需要继续进行下一轮优化。

------

### 输入格式：

第一行输入一个整数 `n`，表示三地址码指令条数。

接下来 `n` 行，每行一条三地址码指令。

每条指令只可能是以下三种形式之一：

```
x = y op z
x = y
print x
```

其中：

```
op ∈ {+, -, *, /}
```

------

### 输出格式

输出优化后的三地址码程序，每条指令占一行，输出格式必须严格符合以下规范：

赋值指令：x = y

二元运算指令：x = y op z

输出指令：print x

输出中不需要输出指令条数。

为了避免不同优化顺序导致答案不同，本题规定：

1. 必须按指定优化顺序执行；
2. 必须重复优化直到代码不再变化；
3. 不允许改变未删除指令之间的相对顺序；
4. 不允许重命名变量；
5. 不允许引入新的临时变量；
6. 不需要处理跨基本块优化；
7. 本题中整个程序视为一个基本块；
8. 输出格式必须符合题目要求。
